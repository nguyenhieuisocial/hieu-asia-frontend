## FIXED — Security (highest priority)

- **[CRITICAL] Generic admin proxy enforced NO role** — any `viewer` could run owner-only
  mutations (SePay refunds, comped paid-access grants, RAG reindex, task retry) because the
  backend only checks the shared admin token and treats the role as decorative.
  → `admin-proxy/[...path]/route.ts`: added a per-user role gate (GET=viewer, writes=admin,
  secrets/refunds/paid-access=owner).
- **[HIGH] Secrets reveal/rotate/delete bypassed roles** — same root cause; secrets now owner-only.
- **[HIGH] Payout CSV (bank-account numbers + PII)** had no role gate → now `requireAdminSession('admin')`.
- **[MED] Open redirect via `?next=`** (`//evil.com` passed) → added `safeNextPath()` same-origin
  guard in `auth.ts`, used by middleware + login.
- **[LOW] Re-orchestrate (paid pipeline re-run)** was viewer-runnable → now admin-only.
- **[MED] Sensitive proxied mutations (grant/revoke access, rename, retry)** — covered by the
  proxy role gate above.
